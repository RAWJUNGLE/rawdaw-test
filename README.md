# RAWDAW Desktop

Нативная десктопная сборка RAWDAW для **macOS (Apple Silicon arm64 и Intel x64), Windows (x64) и Linux (x64)**. Ядро тяжёлых аудио-алгоритмов вынесено в **C++**, а интерфейс — это ваш исходный `RAWDAW_v1_0.html` без визуальных изменений.

## Что внутри

Приложение собрано на **Electron**. Это сознательный выбор: Electron несёт с собой один и тот же движок Chromium на всех четырёх платформах, поэтому Web Audio, AudioWorklet, WASM и весь DOM ведут себя идентично везде. Альтернатива (Tauri) использует системные webview, которые на каждой ОС разные и дают расхождения в аудио — для DAW это неприемлемо.

Структура проекта:

```
RAWDAW-Desktop/
├─ package.json              корневой манифест Electron + конфиг electron-builder
├─ src/main/
│  ├─ main.js                главный процесс: грузит C++ ядро, окно, IPC
│  └─ preload.js             contextBridge → window.RawCore (безопасный мост)
├─ renderer/
│  ├─ index.html             ВАШ DAW (CDN-скрипты заменены на локальные)
│  ├─ native-hook.js         аддитивный хук, подключающий C++ BPM
│  └─ vendor/                локальные jszip + FileSaver (офлайн-работа)
├─ native/rawcore/           C++ ядро (нативный аддон Node N-API)
│  ├─ src/                    bpm.*, stretch.*, addon.cpp
│  ├─ vendor/                 BeatDetektor, Signalsmith (как есть, без правок)
│  ├─ binding.gyp            конфиг сборки нативного модуля
│  └─ test/test-bpm.js       тест ядра из Node
├─ build-resources/          entitlements для macOS (подпись/нотаризация)
└─ .github/workflows/build.yml   CI: собирает все 4 платформы
```

## C++ ядро (`rawcore`)

Ядро — это нативный модуль Node (N-API), который грузится в главном процессе Electron и доступен в интерфейсе как `window.RawCore`. В него вынесены два алгоритма из присланных вами библиотек:

**1. Определение BPM** (`src/bpm.cpp`). Гибридный детектор: основная оценка темпа — автокорреляция огибающей онсетов (спектральный поток через ваш `FFT.h`), а кросс-проверка — алгоритм **BeatDetektor** (CubicFX), запущенный точно по эталонному `main.cpp` (FFT → collapse → `process` → `contest` → `run`). Это новая функция: раньше BPM в проекте можно было считать только встроенным JS-портом, теперь тот же расчёт делает C++.

Результат на синтетических тестах (клик-треки 90–174 BPM, 44.1k и 48k): **все темпы определяются точно, с точностью до десятых, в правильной октаве**.

**2. Тайм-стретч с маркерами** (`src/stretch.cpp`). Офлайн-рендер через **Signalsmith Stretch** (как есть): равномерное растяжение по коэффициенту, сдвиг высоты в полутонах и режим warp-маркеров (пары inputBeat/outputBeat — посегментное растяжение, логика повторяет `computeStretchSegments` из вашего рендерера). Доступен как `window.RawCore.stretchOffline`.

### Принцип «UI не меняется»

Хук `renderer/native-hook.js` — единственное добавление в интерфейс, и оно **не создаёт ни одного видимого элемента**. Когда приложение работает в Electron, хук перенаправляет только число BPM на C++ ядро, оставляя транзиенты, сетку warp-маркеров, кэш и сворачивание октав абсолютно такими же, как в оригинале. В обычном браузере (без `window.RawCore`) файл вообще ничего не делает, и работает ваш исходный JS-детектор.

Реалтайм-стретч и экспорт по-прежнему идут по существующему WASM-пути Signalsmith — это сохраняет идентичное поведение. Нативный `stretchOffline` готов и проверен, его подключение в путь экспорта — отдельный опциональный шаг (см. `ARCHITECTURE.md`), потому что он затрагивает код экспорта и потенциально меняет результат.

## Сборка

Требуется **Node.js 20+**, **Python 3** и компилятор C++ (g++/clang/MSVC). На каждой платформе:

```bash
npm install            # ставит зависимости, собирает C++ ядро под ABI Electron
npm run test:native    # быстрый тест ядра из Node (BPM + стретч)
npm start              # запуск приложения
```

Сборка установщиков:

```bash
npm run dist:mac       # .dmg + .zip для arm64 и x64  (только на macOS)
npm run dist:win       # .exe (NSIS) для x64          (только на Windows)
npm run dist:linux     # .AppImage + .deb для x64      (на Linux)
```

`npm install` через `postinstall` сам собирает `rawcore.node` и пересобирает его под ABI Electron (`@electron/rebuild`).

> Замечание для Linux-сборки в изолированном окружении: если node-gyp не может скачать заголовки Node, укажите системные — `npm --prefix native/rawcore run rebuild -- --nodedir=/usr`. На обычной машине это не нужно.

## Сборка всех платформ сразу

macOS-бинарники можно собрать только на macOS, Windows — только на Windows (это ограничение подписи и нативной компиляции, не проекта). Поэтому в репозитории есть готовый GitHub Actions — `.github/workflows/build.yml`. Запушьте проект в репозиторий GitHub: workflow на трёх раннерах (macOS-14, windows-latest, ubuntu-22.04) соберёт все четыре цели и выложит установщики как артефакты. Тег вида `v1.0.0` дополнительно создаёт черновик релиза со всеми файлами.

## Подпись и нотаризация

Готовые к распространению сборки нужно подписать вашими собственными сертификатами — у меня их нет и быть не может:

- **macOS**: нужен Apple Developer ID. Передайте в CI секреты `CSC_LINK` (ваш .p12 в base64), `CSC_KEY_PASSWORD`, и для нотаризации `APPLE_ID`, `APPLE_APP_SPECIFIC_PASSWORD`, `APPLE_TEAM_ID`. Entitlements уже настроены (`build-resources/entitlements.mac.plist`) — там разрешён JIT для WASM и доступ к микрофону.
- **Windows**: сертификат Authenticode. Секреты `WIN_CSC_LINK` и `WIN_CSC_KEY_PASSWORD`. Без подписи .exe соберётся, но SmartScreen будет предупреждать.
- **Linux**: подпись не требуется.

## Что сделано в этой сессии, а что требует вашей машины

**Готово и проверено здесь:**
- C++ ядро BPM — собрано, проходит все тесты (12/12), точность до десятых.
- C++ ядро стретча — собрано, проверены равномерный/питч/маркеры режимы.
- Нативный аддон N-API — собран, протестирован из Node (`npm run test:native` = ALL PASSED).
- Electron-оболочка, preload-мост, аддитивный хук — написаны, синтаксис проверен.
- Рендерер — CDN-скрипты заменены на локальные, хук подключён, видимый UI не тронут.
- Конфиги electron-builder (dmg/nsis/AppImage/deb) и CI на 4 платформы.

**Требует вашей машины или CI** (в изолированной среде нельзя скачать рантайм Electron — релизные ассеты GitHub отдают `host_not_allowed`):
- Первый запуск GUI (`npm start`) и пересборка ядра под ABI Electron.
- Сборка установщиков под каждую ОС.
- Подпись и нотаризация вашими сертификатами.

## Присланные библиотеки: статус

- **beatdetektor-audiolab (BPM)** — интегрирован в ядро.
- **signalsmith-stretch** — интегрирован в ядро (стретч).
- **superduper-dsp** — это эффекты на **Rust** в формате CLAP-плагина. Это отдельный формат сборки, не часть ядра DAW; путь интеграции описан в `ARCHITECTURE.md`, но в рамки этого порта он не входит.
- **Just-a-Sample** — инструмент на **JUCE C++** (тоже отдельный плагин с собственным CMake-проектом). Аналогично — описан, но вне рамок порта ядра.

Подробности об архитектуре и интеграции — в `ARCHITECTURE.md`.
